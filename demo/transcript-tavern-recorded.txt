# The tavern is a small text-adventure world. One file builds it:
# - a `tuplespace` named `world` seeded with room + menu tuples
# - an `object barkeep` with two facets: `customer` and `staff`
# - a `grammar` that parses player input
# - a `dispatch` lambda that takes (actor, cmd) and routes the
# command through whatever facet the actor presents

# Capability discipline rides on the ACTOR, not on the parse. The
# same parsed `:StockCount` command lands through `staff` (returns
# the stock number) and is rejected through `customer` (runtime
# error). That's the demo's punchline.

3. Two facets, two actors
────────────────────────────────────────────────────────────────────────

# source: the barkeep object — customer and staff facets
```fmpl
object barkeep {
  .#facets
    customer: [greet, menu, order]
    staff:    [restock, stock_count]
  .#public
  init(): self.stock = 10

  greet(name): "Welcome to The Rusty Flagon, " + name + "!"
  menu(): "We serve: ale (3g), mead (5g), stew (7g)"
  order(item): "Here is your " + item + ". Enjoy!"
  restock(n): self.stock = self.stock + n
  stock_count(): self.stock
  stock: 10
}
```
# source: binding the two actor handles
```fmpl
let customer = barkeep.as(:customer)
let staff    = barkeep.as(:staff)
```
# `barkeep.as(:customer)` returns a sealed handle that only exposes greet/menu/order. `barkeep.as(:staff)` exposes restock/stock_count instead. Same underlying object; two restricted views.

7. Grammar parses a player command
────────────────────────────────────────────────────────────────────────

# source: the command grammar
```fmpl
let cmd_grammar = grammar CmdGrammar {
  command  = look | talk | order | stock
  look     = "look"          => :Look
  talk     = "talk to " name:n => [:Talk, n]
  order    = "order " item:i => [:Order, i]
  stock    = "stock"         => :StockCount
  name     = [a-zA-Z ]+:cs => cs
  item     = [a-z]+:cs => cs
}
```
# The same grammar parses both customer verbs (look/talk/order) and a staff verb (stock). Capability isn't a property of the parse — it's a property of who's dispatching.
[tavern] fmpl> "stock" @ cmd_grammar.command
=> :StockCount
@captured parsed_stock_cmd = :StockCount

────────────────────────────────────────────────────────────────────────
8. Dispatcher — facet-NEUTRAL, takes the actor as an argument
────────────────────────────────────────────────────────────────────────

# source: the dispatcher routes through actor.METHOD()
```fmpl
let dispatch = \actor \cmd cmd @ {
  :Look           => world.rd(:room).data.desc
  [:Talk, npc]    => actor.greet(npc)
  [:Order, item]  => actor.order(item)
  :StockCount     => actor.stock_count()
  _               => "You can't do that."
}
```
# Notice the dispatcher uses `actor.greet(npc)`, `actor.stock_count()` — it doesn't know or care which facet was passed. The actor's facet decides whether the call lands.

────────────────────────────────────────────────────────────────────────
9. Dispatch :StockCount as the BARKEEP — landed
────────────────────────────────────────────────────────────────────────

# Bind the parsed command, then dispatch it through the staff actor. `actor.stock_count()` resolves to the staff facet's stock_count method. The barkeep can do this.
[tavern] fmpl> let parsed = "stock" @ cmd_grammar.command
=> :StockCount
[tavern] fmpl> dispatch(staff)(parsed)
=> 10
