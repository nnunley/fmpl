## Revisions vs. R1 draft

**Big realization (implementer, not PAR — investigating R-J-S-1):** `Lambda` in fmpl-core stores `code: Arc<CompiledCode>` (bytecode), NOT an AST. Pretty-printing bytecode back to source is information-lossy — operator precedence is baked into stack ordering; you can't recover the original syntactic form. The "AST-to-FMPL-source pretty-printer" framing is **structurally wrong** for the runtime artifacts we have today. Lambdas synthesize cleanly only when we have AST-level source (which `eval()` does, just before compile).

This collapses the synthesizer question: there's nothing meaningful to "synthesize" from a bytecode-only Lambda. The roadmap's synthesizer was speculative — designed before the post-ITER-0004 list-form AST was a value-level entity.

**Revised iteration scope: drop the synthesizer entirely from 0005b.** Defer constructor synthesis to a future iteration (call it ITER-0005b-SYNTH) when a real consumer exists. Per `feedback_ship_infrastructure_with_first_consumer.md`: don't ship infrastructure without consumers.

What we ship in 0005b:
- T1: `Hash::compute` (location moved per R-I-S-2)
- T2: `SourceStore` API (compact() implementation simplified per R-J-C-1)
- T3: source plumbing into `CompiledCode::save_to_store` ONLY (per R-I-S-1; ParseState + ObjectDb stay unchanged)
- T4: NEW `recover_incompatible()` loader pass (per R-I-C-1) + SCENARIO-0102 evidence
- T5: SCENARIO-0100 evidence (same-source-deduplicates)
- T6: Wrap artifacts

**Deferred to follow-ups:**
- Constructor synthesis for lambda/object/grammar (AC-4, AC-5, SCENARIO-0101) → ITER-0005b-SYNTH, when a real consumer demands sourceless-artifact recovery
- ObjectDb source-threading design (R-J-S-2; needs design call) → ITER-0005b-OBJ
- AC-7 GC keyspace-scan orchestration → ITER-0005b-GC (compact() primitive ships in T2)

This split aligns with the Discord-bot demo timing constraint: the bot's CompiledCode paths all flow through `eval()` so source IS available; the synthesizer fallback isn't load-bearing.

### Seam A — Hash::compute placement (R-I-S-2 fix)

Add `Hash::compute(bytes) -> Hash` as a free function in `fmpl-persistence/src/hash_compute.rs` (NEW module). fmpl-types stays carrier-only:

```rust
// fmpl-persistence/src/hash_compute.rs
use fmpl_types::Hash;

/// Compute the blake3 hash of `bytes`. Lives in fmpl-persistence
/// because SourceStore is the first (and currently only) consumer;
/// fmpl-types stays a zero-impl carrier-only crate.
pub fn hash_bytes(bytes: &[u8]) -> Hash {
    Hash::from_bytes(*blake3::hash(bytes).as_bytes())
}
```

Re-export from `lib.rs`. blake3 is already a fmpl-persistence dep — no Cargo.toml change.

### Seam B — SourceStore in fmpl-persistence (R-J-C-1 + simplified)

`fmpl-persistence/src/source_store.rs` (NEW module):

```rust
use crate::fjall_backend::FjallStore;
use crate::hash_compute::hash_bytes;
use crate::{Store, StoreError};
use fmpl_types::Hash;
use std::collections::HashSet;
use std::path::Path;

pub struct SourceStore {
    fjall: FjallStore,
}

impl SourceStore {
    /// Open a SourceStore at `path` (creates the dir if absent).
    pub fn open(path: impl AsRef<Path>) -> Result<Self, StoreError> {
        Ok(Self {
            fjall: FjallStore::open(path)?,
        })
    }

    /// Insert `bytes`. Returns the content-hash. Idempotent.
    pub fn put(&self, bytes: &[u8]) -> Result<Hash, StoreError> {
        let h = hash_bytes(bytes);
        self.fjall.insert(h.as_bytes(), bytes)?;
        Ok(h)
    }

    /// Retrieve bytes by hash. None if absent.
    pub fn get(&self, hash: Hash) -> Result<Option<Vec<u8>>, StoreError> {
        self.fjall.get(hash.as_bytes())
    }

    /// Remove source entries whose hash is NOT in `referenced`.
    /// Uses fjall::Keyspace::remove directly — does NOT extend the
    /// Store trait (per pre-iter PAR R-J-C-1).
    pub fn compact(
        &self,
        referenced: impl IntoIterator<Item = Hash>,
    ) -> Result<CompactStats, StoreError> {
        let keep: HashSet<[u8; 32]> = referenced.into_iter().map(|h| h.into_bytes()).collect();
        let mut to_remove: Vec<Vec<u8>> = Vec::new();
        for item in self.fjall.iter() {
            let (key, _value) = item?;
            if key.len() == 32
                && let Ok(arr) = <[u8; 32]>::try_from(&key[..])
                && !keep.contains(&arr)
            {
                to_remove.push(key);
            }
        }
        let removed = to_remove.len();
        // Direct fjall::Keyspace::remove via the FjallStore's #[doc(hidden)]
        // keyspace() accessor. The Store trait stays sized to its current
        // surface; only the SourceStore needs delete and only against
        // FjallStore concretely.
        let ks = self.fjall.keyspace();
        for k in to_remove {
            ks.remove(k).map_err(StoreError::from)?;
        }
        let retained = keep.len();
        Ok(CompactStats { retained, removed })
    }
}

pub struct CompactStats {
    pub retained: usize,
    pub removed: usize,
}
```

**Notes:**
- Uses `FjallStore` concretely (not `Box<dyn Store>`). This is OK: SourceStore is a typed wrapper on FjallStore semantics, not a backend abstraction.
- **Future-genericness cost** (per R2 PAR R-K-S-1): making `SourceStore` generic over `Store + Send + Sync` later is a breaking API change, not a no-op. `compact()` requires `Keyspace::remove` which lives on `FjallStore` only — to genericize, either (a) promote `remove` to the `Store` trait (forces every future `Store` impl, including read-only / append-only backends, to implement delete semantics they may not have), OR (b) add a `RemovableStore` supertrait that `SourceStore<S: RemovableStore>` requires. Either path changes downstream signatures. **Name the cost here so the follow-up author doesn't rediscover it.** ITER-0005b-SYNTH and ITER-0005b-OBJ both consume SourceStore directly today — they would also need updating if SourceStore's signature changes.
- `keyspace()` is the `#[doc(hidden)] pub` escape hatch from ITER-0005a.5 R-A-S-1. Original docstring describes it as "tests that need to bypass the envelope writer when seeding synthetic records." SourceStore::compact is NOT a test — this is the escape being repurposed. R2 PAR R-K-S-1 flagged this gap explicitly: when T2 lands, update `fjall_backend.rs`'s `keyspace()` docstring to acknowledge the source-store-compaction use case so future callers don't read the doc and assume the escape is test-only. The underlying reasoning ("don't pollute the trait for one concrete consumer that will always be FjallStore-backed") is defensible; the doc just needs to say so.

### Seam C — `CompiledCode::save_to_store` source plumbing (R-I-S-1 scope-tightened)

```rust
pub fn save_to_store<S: Store>(
    &self,
    store: &S,
    source_store: &SourceStore,
    key: &str,
    source: Option<&[u8]>,
) -> Result<()> {
    use crate::persistence::envelope::write;
    use crate::persistence::schema::PayloadKind;
    use fmpl_types::Hash;

    let source_hash = match source {
        Some(bytes) => source_store.put(bytes).map_err(|e| {
            Error::BytecodePersistenceError(format!("source store put: {e}"))
        })?,
        None => Hash::NONE, // Synthesis deferred to ITER-0005b-SYNTH
    };

    write(store, key.as_bytes(), self, PayloadKind::CompiledCode,
          crate::VM_VERSION, source_hash)
        .map_err(|e| Error::BytecodePersistenceError(e.to_string()))?;
    Ok(())
}
```

**Out of scope this iteration:**
- `ObjectDb::save_to_store` stays unchanged. The N+1-write design (R-J-S-2) needs a design call; defer to ITER-0005b-OBJ.
- `ParseState::save_to_store` stays unchanged (per R-I-S-1; ParseState has no source).
- `spill_to_store` + `set_memo` stay unchanged.

Call sites that need updating: `CompiledCode::save_to_store` (1 fn) + its tests in `fmpl-persistence/tests/bytecode_persistence.rs` (3-5 test sites). All pass `&source_store` + `Some(source)` for the cases that have source, `None` for unit tests that don't care.

### Seam D — `recover_incompatible()` standalone pass (R-I-C-1)

New function in `fmpl-persistence/src/loader.rs`:

```rust
/// Pass that walks `store`, identifies records whose envelope is
/// well-formed but whose payload is incompatible (SkippedIncompatible
/// or SkippedUnknownKind), and attempts source-store recovery.
///
/// Recovery: resolve the envelope's `source_hash` against `source_store`;
/// if the source is present, hand (key, source_bytes) to the caller's
/// `recompile` closure; if recompile succeeds, the recovered value is
/// counted via the returned `RecoveryStats`. The caller is responsible
/// for any side effects (e.g. binding the recompiled value under `key`
/// in a higher-level VM state) — this function returns stats only.
///
/// The `iter_store` callback contract (fires only on Loaded) is
/// UNCHANGED — recovery is a separate pass, not an extension of iter.
/// Per ITER-0005b pre-iter PAR R-I-C-1.
pub fn recover_incompatible<S, F>(
    store: &S,
    source_store: &SourceStore,
    expected_vm_major: u16,
    mut recompile: F,
) -> Result<RecoveryStats, StoreError>
where
    S: Store + ?Sized,
    F: FnMut(&[u8] /* key */, &[u8] /* source */) -> Result<(), RecoveryError>,
{
    // Walk store.iter(); for each item, call decode(). If outcome is
    // SkippedIncompatible or SkippedUnknownKind, lift the header's
    // source_hash field, look it up in source_store, invoke recompile
    // closure on success. Track stats.
    todo!()
}

pub struct RecoveryStats {
    /// Records that decoded successfully — recovery did nothing.
    pub loaded_passthrough: u32,
    /// Records that failed decode AND had source AND recompile succeeded.
    pub recovered_from_source: u32,
    /// Records that failed decode AND had source AND recompile failed.
    pub recompile_failed: u32,
    /// Records that failed decode AND had no source (source_hash == Hash::NONE).
    pub unrecoverable_no_source: u32,
    /// Records that failed decode AND had source_hash but the source
    /// was missing from the source store.
    pub unrecoverable_source_missing: u32,
    /// Records skipped due to corruption (not recovery-eligible).
    pub skipped_corrupt: u32,
}

#[derive(Debug, thiserror::Error)]
pub enum RecoveryError {
    #[error("recompile failed: {0}")]
    Recompile(Box<dyn std::error::Error + Send + Sync>),
}
```

**Notes:**
- The closure signature `FnMut(&[u8], &[u8]) -> Result<(), RecoveryError>` is the seam where the consumer (fmpl-core's `eval()` driver) plugs in. The function doesn't know about VMs or compilation — it stays in fmpl-persistence.
- `RecoveryStats` is NEW, separate from `LoaderStats`. Doesn't disturb `LoaderStats::check_invariants` (R-I-C-1's concern about invariant fit).
- Header's `source_hash` field is already in the envelope (set by Seam C); `recover_incompatible` reads it via the existing `EnvelopeHeader` zerocopy decode.

## Risks

- **recover_incompatible's closure plug-in pattern is new.** No prior FMPL code uses a "decode-then-recompile" pattern; the closure shape may need iteration during T4. Mitigation: keep the closure narrow (key bytes + source bytes in, Result in), defer any state-binding concerns to the caller.
- **Source store grows unboundedly without GC.** AC-7 keyspace-scan orchestration is deferred. SourceStore::compact() ships in T2 so the primitive is available; callers wire up the scan later. Documented in progress.md.
- **SCENARIO-0100 needs `eval()` to derive source consistently.** The source bytes passed to `save_to_store` must match exactly across two `eval()` calls of the same input. Verify `eval()` doesn't apply any input normalization (whitespace trimming, comment stripping) before persisting; if it does, the dedup story needs adjustment.

## Out of scope (deferred to explicit follow-up iterations)

- **ITER-0005b-SYNTH**: Constructor synthesizer for lambda/object/grammar (AC-4, AC-5, SCENARIO-0101). Requires either lambda body to carry AST (not bytecode) — possible future Lambda struct change — OR a different synthesis story (e.g. attach an `Option<source>` field to runtime artifacts that DID come from source). Defer until a real consumer demands sourceless recovery.
- **ITER-0005b-OBJ**: ObjectDb source-threading design (R-J-S-2). Decide per-object vs whole-DB source; loop integration if per-object.
- **ITER-0005b-GC**: AC-7 GC keyspace-scan orchestration. SourceStore::compact() primitive ships in T2 here; caller-side keyspace scanning is the GC piece.
